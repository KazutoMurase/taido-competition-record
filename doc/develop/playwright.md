# 大会スケジュールの自動進行

指定したコートの現在競技を、点呼から全試合の記録まで自動で進める。
現在競技が終了し、次の競技へ移動したことを確認すると停止する。

このツールはローカル環境の大会データを実際に更新する。
実行前に、`.env`の`COMPETITION_NAME`と対象コートが正しいことを必ず確認する。
本番環境に対しては使用しないこと。

PlaywrightはDockerコンテナ内で実行されるため、ホストへのPlaywrightやChromiumのインストールは不要である。

## 基本的な使い方

最初にアプリを起動する。

```bash
docker compose up
```

別のターミナルで、リポジトリルートから進めたいコートを指定する。
以下はAコートの現在競技を進める例である。

```bash
tools/advance-schedule.bash A
```

実行中は、対象競技、試合番号、登録した結果が表示される。

```text
[advance-schedule] court=A
[advance-schedule] current schedule=1, game=1, competition=男子法形
[advance-schedule] attendance completed
[advance-schedule] recorded game=1, result=赤旗2本
[advance-schedule] recorded game=2, result=赤旗1本
[advance-schedule] completed 男子法形; next schedule=2
```

点呼や一部の試合がすでに完了している場合は、残りの処理から続行する。

## 複数の競技を順番に進める

複数のコートを指定すると、指定した順番に各コートの現在競技を1つずつ進める。

```bash
tools/advance-schedule.bash A B A C B
```

この例では、A、B、A、C、Bの順に処理する。
途中で失敗した場合は、その場で停止する。

### コートごとに並列実行する

`--parallel`を指定すると、異なるコートを並列に処理する。
同じコートの競技は、指定した順番どおりに処理される。

```bash
tools/advance-schedule.bash --parallel A B A C B
```

この例ではA、B、Cコートを並列に処理し、各コート内では指定順を維持する。
ログの先頭には`[court A]`のようにコート名が表示される。

いずれかのコートで失敗しても、実行中のほかのコートは処理を続ける。
すべてのコートが終了した後、1つでも失敗していればコマンド全体がエラーで終了する。

## 計画ファイルを使う

実行順が長い場合は、計画ファイルを使用する。

```bash
tools/advance-schedule.bash --file e2e/advance-plan.txt
```

コートごとに並列実行する場合は`--parallel`を追加する。

```bash
tools/advance-schedule.bash --parallel --file e2e/advance-plan.txt
```

計画ファイルには、1行につきコートと現在競技IDを記述する。

```text
# court schedule_id
A 1
B 1
A 2
C 1
B 2
```

空行と`#`以降は無視される。
サンプルは`e2e/advance-plan.example.txt`にある。

現在競技IDは省略できる。

```text
A
B
A
```

長い計画では現在競技IDを指定することを推奨する。
途中で失敗して同じファイルを再実行した場合に、完了済みの処理を安全にスキップできるためである。

| コートの現在競技ID | 動作                               |
| ------------------ | ---------------------------------- |
| 計画と同じ         | 現在競技を実行する                 |
| 計画より先         | 完了済みとしてスキップする         |
| 計画より前         | 計画と状態が一致しないため停止する |

## 自動で入力される内容

点呼が未完了の場合は、全選手または全団体を出席として登録する。
その後、競技が終了するまでランダムな試合結果を入力する。

| 競技種別                     | 入力内容                   |
| ---------------------------- | -------------------------- |
| 個人・団体法形のトーナメント | 赤旗0〜3本                 |
| 個人・団体実戦               | 赤勝利または白勝利         |
| 団体法形                     | 主審・副審得点と減点       |
| 展開                         | 時間、主審・副審得点、減点 |

入力後は保存された結果を確認し、正しく保存されていない場合は停止する。
団体法形・展開の予選では、全試合の終了後に結果を確定して決勝進出団体を反映する。

## 失敗した場合

計画ファイルに現在競技IDを記載している場合は、問題を解消して同じコマンドを再実行する。
完了済みの競技はスキップされ、未完了の競技から続行する。
エラーの内容はコンソールに表示される。

## 注意事項

- 複数のターミナルから同じコートを同時に実行しない。
- 実行中は対象コートを手動で操作しない。
- 処理済みの試合結果は、ツールを再実行しても元に戻らない。
- このツールはローカル環境専用であり、本番環境やCloud Runには使用しない。
